# BKey Developer Docs

BKey provides biometric approval infrastructure so that AI agents can request human authorization before taking sensitive actions.

## What is BKey?

BKey is an **OAuth 2.1 + CIBA identity provider** that enables:

- **Agents** to authenticate and request approvals via standard OAuth flows
- **Humans** to approve actions with facial biometrics on their mobile device
- **Developers** to integrate biometric approval into any application

## The Core Primitive: CIBA

**CIBA (Client-Initiated Backchannel Authentication)** is BKey's foundation. Every capability is built on top of it:

1. Your agent authenticates with BKey using **client credentials**
2. Agent calls `bkey.approve("Deploy to production")`
3. BKey sends a **push notification** to the user's mobile app
4. User reviews and approves with **facial biometrics**
5. Agent receives a short-lived **EdDSA JWT** proving consent

The approval token can be used as proof-of-consent in your own app, or passed to BKey's built-in capabilities:

| Capability | Built on CIBA | Description |
|-----------|---------------|-------------|
| **Approval** | Core primitive | Per-action biometric authorization for any operation |
| **Checkout** | `approve:payment` scope | Agent-initiated purchases with amount/merchant approval |
| **Vault** | `approve:secrets` scope | Encrypted secret storage with biometric access gate |
| **Signing** | `approve:action` scope | Document and transaction signing with biometric proof |

## SDKs

Install the SDK for your language:

<CodeGroup>
```bash npm
npm install @bkey/sdk
```

```bash pip
pip install bkey-sdk
```
</CodeGroup>

Go and Rust SDKs are coming soon. See the [GitHub repo](https://github.com/bkeyID/bkey) for updates.
